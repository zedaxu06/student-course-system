请你帮我们设计一个 C 语言实训项目预案，项目名称为《学生选课及学籍管理系统》。

背景：
我们是大一学生，要完成一个 C 语言综合实训项目。项目要求使用 C 语言实现，采用多文件结构，每个功能模块单独写成 .c 文件，公共结构体、宏定义和函数声明写在 .h 文件中。系统需要使用结构体数组或链表、文件读写、菜单交互、输入合法性校验，并具备一定安全性。

请你按照以下要求帮我们设计完整预案：

一、项目简介
说明本系统要解决什么问题，面向哪些用户，主要实现学生信息管理、课程信息管理、学生选课、退课、查询、统计等功能。

二、需求分析
请从功能性需求和非功能性需求两个方面分析。
功能性需求至少包括：
1. 登录模块：管理员登录、学生登录。
2. 学生信息管理：添加学生、删除学生、修改学生信息、查询学生信息、显示全部学生。
3. 课程信息管理：添加课程、删除课程、修改课程信息、查询课程、显示全部课程。
4. 选课管理：学生选课、退课、查看个人已选课程。
5. 学籍管理：维护学生学号、姓名、性别、班级、专业、年级、学籍状态等信息。
6. 统计功能：统计某门课程选课人数、某学生选课数量、按班级或专业统计学生人数。
7. 文件保存与读取：学生信息、课程信息、选课记录都要保存到文件中，程序重新运行后数据不能丢失。

非功能性需求包括：
1. 菜单清晰，操作简单。
2. 输入数据要进行合法性校验，例如学号不能重复、课程号不能重复、选课人数不能超过课程容量。
3. 文件读写要保证数据一致性。
4. 代码结构清晰，便于分工合作和后期维护。

三、总体设计
请设计系统功能模块图，并用文字说明各模块关系。
建议模块包括：
1. main.c：主菜单与程序入口。
2. student.c / student.h：学生信息管理模块。
3. course.c / course.h：课程信息管理模块。
4. select.c / select.h：选课退课模块。
5. file.c / file.h：文件读写模块。
6. login.c / login.h：登录与权限管理模块。
7. common.h：公共结构体、宏定义和全局常量。

四、数据结构设计
请设计以下结构体：
1. Student：学生信息，包括学号、姓名、性别、班级、专业、年级、学籍状态等。
2. Course：课程信息，包括课程编号、课程名称、授课教师、学分、最大人数、当前人数等。
3. Selection：选课记录，包括学号、课程编号、选课状态等。
请说明每个字段的作用。

五、核心业务规则
请详细说明：
1. 学号唯一。
2. 课程编号唯一。
3. 学生只能选择存在的课程。
4. 课程人数满了不能继续选。
5. 学生不能重复选择同一门课程。
6. 退课时必须已经选过该课程。
7. 删除学生时，需要同步删除该学生的选课记录。
8. 删除课程时，需要同步删除该课程相关选课记录。

六、菜单设计
请设计一个控制台菜单，例如：
1. 管理员登录后：
   - 学生信息管理
   - 课程信息管理
   - 选课记录管理
   - 统计查询
   - 保存并退出
2. 学生登录后：
   - 查看个人信息
   - 查看可选课程
   - 选课
   - 退课
   - 查看已选课程
   - 退出

七、文件设计
请设计文件保存方式：
1. students.dat 或 students.txt：保存学生信息。
2. courses.dat 或 courses.txt：保存课程信息。
3. selections.dat 或 selections.txt：保存选课记录。
请说明程序启动时如何加载文件，退出时如何保存文件。

八、详细设计
请列出主要函数名、函数功能、入口参数和返回值。
例如：
- int addStudent(Student students[], int *studentCount);
- int deleteStudent(Student students[], int *studentCount, Selection selections[], int *selectionCount);
- int addCourse(Course courses[], int *courseCount);
- int selectCourse(Student students[], int studentCount, Course courses[], int courseCount, Selection selections[], int *selectionCount);
- int dropCourse(...);
- void saveStudents(...);
- void loadStudents(...);

九、测试方案
请设计测试用例表，至少包括：
1. 正常添加学生。
2. 添加重复学号学生。
3. 正常添加课程。
4. 添加重复课程号。
5. 学生正常选课。
6. 学生重复选同一门课。
7. 课程人数已满时选课。
8. 学生退课。
9. 学生退未选课程。
10. 文件保存后重新启动程序，检查数据是否还在。

十、项目分工
假设小组有 4 人，请给出合理分工：
1. 组长/项目经理：负责整体设计、主菜单、文档整合。
2. 学生模块负责人：负责学生信息管理。
3. 课程与选课模块负责人：负责课程管理、选课退课。
4. 测试与文档负责人：负责测试用例、测试报告、用户手册、PPT。
